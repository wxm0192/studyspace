Docker环境下实现ISCSI外部存储

说明：

iSCSI（Internet Small Computer System Interface），Internet小型计算机系统接口，又称为IP-SAN，该技术是将现有SCSI接口与以太网络(Ethernet)技术结合，基于 TCP/IP的协议连接iSCSI服务端（Target）和客户端(Initiator)，使得封装后的SCSI数据包可以在通用互联网传输，最终实现iSCSI服务端映射为一个存储空间提供给已连接认证后的客户端。

 

步骤01：网络测试

 

本实验需要访问公网环境，测试能够连通公网 。

 

在此通过测试 8.8.8.8 公网地址验证能否连通公网 。

 

在命令行输入 ：

 

#ping 8.8.8.8

 

如果看到如下显示， 说明能够连通公网 ：

 

PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.

 

64 bytes from 8.8.8.8: icmp_seq=3 ttl=110 time=42.7 ms

 

64 bytes from 8.8.8.8: icmp_seq=4 ttl=110 time=42.7 ms

 

64 bytes from 8.8.8.8: icmp_seq=5 ttl=110 time=42.7 ms

步骤04：启动iSCSI容器

 

1、后台启动ISCSI容器

 

 #docker run \

           --detach \

           --name stgt \

           --restart=unless-stopped \

           --net host \

           --privileged \

           --volume /etc/target:/etc/target \

           --volume /run/lvm:/run/lvm \

           --volume /lib/modules:/lib/modules \

           --volume /sys/kernel/config:/sys/kernel/config \

           --volume /dev:/dev \

           -p 3260:3260 \

           wtnb75/stgt

步骤06：创建target存储

 

1、创建target

 

#docker exec stgt tgtadm --mode target --op new --tid 1 --targetname iqn.2017-04.com.comsys:iscsi.fcdisk

 

2、创建lun

 

#docker exec stgt tgtadm --mode logicalunit --op new --tid 1 --lun 1 --backing-store /dev/vdb

（查看到宿主机提供iscsi容器使用的磁盘盘符为vdb

[root@iZ8vbh9av10q37dd842qvsZ ~]# lsblk

NAME   MAJ:MIN RM SIZE RO TYPE MOUNTPOINT

vda    253:0    0  40G  0 disk

└─vda1 253:1    0  40G  0 part /

vdb    253:16   0  20G  0 disk）

3、授权

 

#docker exec stgt tgtadm --mode target --op bind --tid 1 --initiator-address 172.30.2.0/24

(宿主机的IP网段为172.30.2.0/24)

#docker exec stgt tgtadm --mode target --op bind --tid 1 --initiator-address 172.17.2.0/24

(容器的IP网段为172.17.2.0/24)

 

4、查看已经定义的信息

 

#docker exec stgt tgtadm --mode target --op show

步骤07：客户端发现并登录存储

 

1、安装客户端软件（虚拟机已经安装，无需做此步骤）

 

# yum install iscsi-initiator-utils

 

2、启动iscsi服务

 

 #service iscsi start;service iscsid start

（查看iSCSI服务的状态，#service iscsi start;service iscsid status）

3、发现iSCSI存储

 

# iscsiadm -m discovery -t st -p 172.30.2.191

172.30.2.191:3260,1 iqn.2017-04.com.comsys:iscsi.fcdisk

 

4、登录iSCSI存储

 

# iscsiadm -m node   -d -T   iqn.2017-04.com.comsys:iscsi.fcdisk   -p 172.30.2.191 -l

Logging in to [iface: default, target: iqn.2017-04.com.comsys:iscsi.fcdisk, portal: 172.30.2.191,3260]

Login to [iface: default, target: iqn.2017-04.com.comsys:iscsi.fcdisk, portal: 172.30.2.191,3260] successful.
